## Prompt

## Project Identity
**Name:** Todo Console App - Phase I
**Type:** Command-line Application
**Language:** Python 3.13+

---

## Core Principles

### 1. Simplicity First
- Keep code simple and readable
- One function = one responsibility
- Avoid over-engineering
- No external libraries unless absolutely necessary

### 2. Spec-Driven Development
- All code must be generated from specifications
- Manual coding is NOT allowed
- Specifications must be clear and detailed
- Iterate on specs, not code

### 3. User-Centric Design
- Clear prompts and messages
- Helpful error messages
- Confirm destructive actions (delete)
- Intuitive menu navigation

---

## Technology Constraints

### Required Stack
- **Language:** Python 3.13 or higher
- **Storage:** In-memory only (Python list/dict)
- **Interface:** Command-line (terminal/console)
- **Dependencies:** Standard library only (no pip packages)

### Forbidden in Phase I
- ‚ùå Databases (SQLite, PostgreSQL, etc.)
- ‚ùå Web frameworks (Flask, FastAPI, Django)
- ‚ùå File persistence (JSON, CSV, pickle)

## Code Standards

### Structure
- Single file: `src/main.py`
- Functions for each operation
- Clear separation of concerns
- Main menu loop at bottom

### Naming Conventions
- Functions: `snake_case` (e.g., `add_task`, `view_tasks`)
- Variables: `snake_case` (e.g., `task_list`, `user_input`)
- Constants: `UPPER_CASE` (e.g., `MAX_TASKS`)

### Documentation
- Each function must have a docstring
- Explain complex logic with inline comments
- README must include setup and run instructions

### Error Handling
- Validate user input
- Handle invalid menu choices gracefully
- Catch and display errors clearly
- Never crash - always recover

---

## Data Model

### Task Structure
Each task is a Python dictionary with:
```python
{
    "id": int,           # Unique auto-incrementing ID
    "title": str,        # Required, max 200 chars
    "description": str,  # Optional, max 1000 chars
    "completed": bool    # Default: False
}
```

### Storage
- All tasks stored in a Python list: `tasks = []`
- IDs start at 1 and auto-increment
- No persistence - data lost on exit

---

## User Experience Rules

### Menu System
- Display numbered options clearly
- Show current state (e.g., "3 tasks pending")
- Allow easy exit (option 6)
- Loop continuously until user exits

### Input Validation
- Reject empty titles
- Accept empty descriptions
- Validate task IDs exist
- Handle non-numeric input

### Feedback
- Confirm every action ("Task added!", "Task deleted!")
- Show what was changed
- Use clear, friendly language

### Formatting
- Use separators for readability (===, ---)
- Show task status: [Pending] or [Completed]
- Number tasks consistently
- Keep output clean and scannable
